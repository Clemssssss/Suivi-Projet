Guide utilisateur

Manuel Suivi4Me

Cette page présente le fonctionnement du dashboard commercial, les droits par profil, les filtres globaux et les usages recommandés pour suivre la performance, le pipe et les projets.

1 Se repérer

Le header donne accès aux exports, au manuel et au statut de session selon votre rôle.

2 Filtrer

La barre sticky pilote année, champ date, mode CA, énergie, période et mesure.

3 Décider

Les KPI, graphes métier et synthèses textuelles servent de base au pilotage commercial.

Architecture de la page

Header

Le header regroupe les actions disponibles selon le rôle connecté : exports, email, manuel, import ou administration.

Filtres globaux

Ils pilotent la vue courante et impactent KPI, graphes, analyses textuelles, PDF et email.

Blocs métier

Les sections “Prise d’affaires / Performance” et “Pipe commercial” donnent la lecture métier la plus directe.

Filtres et lecture métier

Année commerciale

Quand une année est choisie, les calculs se concentrent dessus. En mode “Toutes les années”, le dashboard garde tout le périmètre filtré visible.

Champ date

Il définit la date qui sert de référence pour l’année et les analyses temporelles. Vérifie toujours ce point avant d’interpréter un graphique de période.

Période

Les périodes glissantes et trimestres permettent d’isoler des fenêtres courtes sans casser le reste du modèle métier.

Droits par profil

Consultation

Profil lecture seule. Peut consulter, lire le manuel et se déconnecter, sans modifier les filtres ni lancer d’action.

User

Profil opérationnel. Utilise le dashboard et ses analyses sans accès à l’administration.

Admin

Profil complet avec import, source SharePoint, logs, whitelist et maintenance des données.

Graphes et analyses

Personnalisation

Certains graphes proposent un bouton 🎛 pour changer le type d’affichage selon l’utilisateur.

Bloc texte

Le texte sous les graphes reprend le périmètre filtré et ajoute des informations liées au graphe lui-même : top catégorie, série dominante, volume visible.

Bonnes pratiques

  • Faire un hard refresh après déploiement ou changement majeur.
  • Vérifier le profil connecté avant de conclure qu’une action “manque”.
  • Contrôler le champ date avant de lire les années, périodes et trimestres.
  • Si la base dataset est indisponible, la session peut désormais restaurer la dernière copie chargée.
